# Code of Conduct

## Our Commitment

RCT Platform is committed to a respectful, safe, and inclusive community for
contributors, maintainers, and users across issues, pull requests, discussions,
documentation, and other project spaces.

We expect all participants to act professionally, assume good intent, and help
keep collaboration constructive even during technical disagreement.

## Expected Behavior

- Use welcoming and respectful language.
- Focus feedback on code, design, tests, security, and documented behavior.
- Accept accountability, corrections, and different technical viewpoints.
- Share context and evidence when challenging an implementation decision.
- Help maintain a repository that is safe for global and cross-cultural use.

## Unacceptable Behavior

The following behaviors are not acceptable in project spaces:

- Harassment, intimidation, discrimination, or personal attacks.
- Sexualized language or imagery.
- Doxxing, threats, stalking, or sustained disruptive behavior.
- Publishing private credentials, internal-only data, or confidential material.
- Trolling, bad-faith review behavior, or intentionally hostile communication.

## Scope

This Code of Conduct applies within all RCT Platform project spaces and when an
individual is representing the project in public contexts.

## Reporting

To report violations or safety concerns, contact:

- <team@rctlabs.co>

Please include links, screenshots, timestamps, and any relevant context. Reports
will be reviewed confidentially and handled as promptly as practical.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, issues, pull
requests, or other contributions that violate this Code of Conduct. Depending on
severity, consequences may include warning, temporary restriction, or permanent
ban from project spaces.
